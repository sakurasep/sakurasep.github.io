字符串

简单理解 String 类

  1. String 类 是 final，不可以被继承
  2. String 类实现了 Serializable 接口，可以被序列化
  3. String 类实现了 Comparable 接口，应该使用 compareTo 方法进行比较
  4. String 类是不可变的，字符串拼接应该考虑 StringBuffer 和 StringBuilder

补充一个知识：在 Java 9 之后 String 数组改用 byte 数组实现（之前是 char 数组），主要是为了优化 String 的占用。

String 类的 方法

  1. hashCode：String 很适合当 HashMap 的键值

  2. substring：常用的字符串截取方法

  3. spilt：按照符号对字符串进行切割

  4. indexOf：用于查找子字符串在原字符串中第一次出现的位置，并返回索引

  5. length：返回字符串长度

  6. isEmpty：判断是否为空

  7. charAt：返回指定索引处的字符

  8. valueOf：将其他类型的数据转换成字符串

  9. getChars: 将字符串复制到字符数组

  10. getBytes: 返回字符串的字节数组

  11. trim：去除字符串两端的空白字符

  12. toCharArray：将字符串转换成字符数组

字符串为什么不可变

  1. String 被 final 修饰，说明没有子类去重写它的方法
  2. String 存在 char 数组中，char 数组也是被 final 修饰，同样的 Java 9 之后使用的 Byte 数组也是同理。

字符串不可变的好处

  1. 保证 String 对象的安全性，避免被篡改
  2. 保证哈希值不会频繁变更，因为 String 类经常会作为哈希表的键值，如果哈希值经常变动，哈希表的性能也会下降。
  3. 可以实现字符串常量池，这样具有相同内容的字符串变量就可以指向同一个字符串对象，节省内存空间。

字符串常量池

首先这一个语句，其实是创建了两个对象。

Java 的逻辑是这样的，首先去字符串常量池看有没有 test这个字符串对象，如果有，就不会在常量池创建这个对象，则是在堆里创建一个对象，然后把地址返回给text。如果没有，则在常量池中创建这个对象，然后继续执行刚才的操作。

所以我们一般创建字符串的时候不使用 new

这样就利用到了 Java 的字符串常量池，如果在常量池中存在这个字符串，就会直接返回这个对象的地址给text，不会额外的在堆上创建一个对象。

补充：字符串常量池在内存的什么位置？

Java 7 之前是在永久代的内存区域中，永久代是 Java 堆的一部分，主要是用来存类信息，方法信息，常量池信息这些静态数据。

Java 7 把常量池从永久代中拿了出来，放在了堆里面。

Java 8 之后取消了大小固定的永久代，用元空间替换了它。

StringBuilder 和 StringBuffer

两者的区别：StringBuffer 是线程安全的。StringBuilder 相反，是为了忽略线程安全为了性能创建出来的。

其余情况两者完全相同。

判断字符串相等

  1. equals 比较的是内容 true
  2. 前者是在堆上的对象地址，后者是在常量池的内存地址 false
  3. 两个都是 new 在堆上的不同对象，一定是 false
  4. 两个都是常量池里的对象 true
  5. a + aa 会优化成 aaa 此时两者都在常量池 true
  6. 前者创建了两个对象，但是最后使用了 intern 去常量池找相关对象，所以与后者同在常量池 true

补充 1：可以使用 Object.equals() 这样不需要判空

补充 2：String 类之间可以使用.contentEquals() 可以将字符串和任何的字符序列进行比较（StringBuffer、StringBuilder）

字符串的拼接

  1. 在循环体内，拼接字符串最好使用 StringBuilder 的 append 方法，因为使用 + 号进行拼接会产生大量的 StringBuilder 对象，占用了更多的内存空间。

  2. concat 也可以进行拼接，只不过他在拼接字符串为 null 的时候，会抛出 NullPointerException

  3. join 也可以进行拼接，这个方法的第一个参数是字符串的连接符。

  4. 还有一个 StringUtils.join()

补充1： Integer 最大值+1，会发生什么

会变成 Integer 的最小值，因为 Java 的整数类型使用的是二进制补码表示法，Integer 的最大值是 0111······ 加一就变成了 1000·····，补码的第一位表示的是符号，所以说就变成了 Integer 的最小值。

补充2： boolean 实际占用几个字节

使用 JOL 工具查看 单独的 boolean 类型或者 boolean 数组，都是 1 个字节。

首先记住的是这个转换规则（自动类型转换）：小转大

byte -> short -> int -> long -> float -> double

char -> int

9. 什么是自动拆箱和装箱

拆箱就是将包装类转换成基本数据类型

装箱就是将基本数据类型转换成包装类

比如说 Integer i = 10; 这里的 i 就是包装类的一个变量。

10. & 和 && 的区别

& 是逻辑与运算

&& 是短路与运算，比如说 条件a && 条件b，如果条件a 是 false，那么条件 b 不会执行判断。

虽然两个都要求左右两边的表达式为 true，总体才能 true，但是短路与会使右边的条件不执行。

比如说 username != null && username.equals(“”) ，反过来写就会导致空指针异常

11. switch 语句能否用到byte、long、String上

这个看 Java 的版本

Java 5 之前只能用于 byte、short、char、int 上

Java 5 新增了枚举 enum

Java 7 可以用于字符串

Java 21 可以用于任意类型，甚至可以添加 when 子句进行复杂判断

12. break continue return 的区别

break：用于循环语句中，主要是中断此次循环并跳出循环

continue：用于循环语句中，主要是跳过此次循环，进行下一次循环

return：直接结束当前的方法，不再执行

13. 用效率最高的方法计算2 * 8

位运算 2 >> 3 相当于乘 2 的三次方

14. 说说自增自减运算

这里主要是看两种情况，第一种是 i++/i– 这种情况下变量先参与运算，运算结束后再进行自增自减

第二种 ++i/–i 这种情况就是变量先进行自增自减，然后参与运算。

15. float是如何表示小数的

二进制浮点数格式，分为三个部分，符号位，指数部分，尾数部分

比如说 25.125

整数部分用二进制表示 11001，小数部分用二进制表示 0.001 合并之后是 11001.001 然后小数点左移 1.1001001 * 2⁴

  1. 符号为 正 = 0
  2. 指数部分 实际要加上 127 所以 4+127=131 二进制 10000011
  3. 尾数部分，去掉整数部分的 1，保留小数部分 1001001 补 0 到 23 位

最后组合起来 0 | 10000011 | 10010010000000000000000
