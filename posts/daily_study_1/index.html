Day1

基础 Java 语法复习

Java 关键字

一、基本概念

  1. abstract：抽象类 抽象方法的修饰词，主要的用法是先抽象一个类以及方法，然后再继承它进行重写实现
  2. class 声明一个类
  3. enum：定义一组固定的常量
  4. null：空值
  5. package：声明类所在的包
  6. void：指定的方法无返回值

二、基本数据类型

  1. boolean：基本数据类型 true false
  2. byte：基本数据类型 八位的有符号整数 范围是 -128 到 127 主要是占用空间小，适用于处理大量数据的场景
  3. char：基本数据类型 用于存储字符
  4. double：基本数据类型 双精度浮点数
  5. float：基本数据类型 单精度浮点数
  6. int：基本数据类型 整数
  7. long：基本数据类型 长整形
  8. short：基本数据类型，短整型

三、控制流

  1. break：跳出当前循环结构 for while switch 语句
  2. switch case 语句：根据某个变量的值 执行不同的代码块
  3. continue：跳过本次循环，进行下一次循环
  4. default：switch case 语句中一定会执行的代码块
  5. do：do 和 while 一般是一起使用的，do 中的代码块一定会执行一次（不管 while 条件是什么）
  6. else：通常和 if 一起使用，算是 if 判断的分支语句
  7. for：for 循环语句

四、类、接口与面向对象

  1. extends：继承一个类或者接口
  2. implements：实现接口
  3. interface：声明一个接口 可以提前写好方法体，等待后续的实现
  4. new：用于创建一个新的对象
  5. super：调用父类的方法
  6. this：引用当前对象

五、访问修饰符

  1. private：访问修饰符，方法和变量只对当前类可见
  2. protected：访问修饰符，方法和变量只对当前包里的类和子类可见
  3. public：访问修饰符，所有类可见

六、修饰符

  1. final：修饰词，三种用法，final 变量表示这个变量是一个不可以修改的常量，final 方法表示这个方法不能够被重写，final 类表示这个类不能够被继承
  2. finally：try catch finally finally 的作用就是这个结构中最后一定会执行的代码块，类似于 switch case 中的 default
  3. native：声明本地方法
  4. static：静态修饰符，静态方法或者静态变量
  5. synchronized：主要是指定多线程代码中的需要同步的部分
  6. transient：修饰的字段不会被序列化
  7. volatile：它所修饰的变量可以被不同线程立刻可见

七、异常处理

  1. try catch：try 中的代码块可以抛出异常，然后 catch 中的代码块负责捕获异常并进行处理
  2. throw：主动抛出异常
  3. throws：声明异常

八、其他关键字与工具

  1. import：引入接口或者类
  2. instanceof：判断某一个对象是不是属于一个类
  3. return：从一个方法中返回值，或者就是提前结束方法的执行

Java 注释

  1. 类、字段、方法 必须使用文档注释
  2. 抽象方法的注释必须符合 Javadoc 规范，最基础包含@param @return
  3. 类注释必须添加创建者和日期@author @date

Java 数据类型

1. 比特和字节

比特是最小的数据单位

1 字节 = 8 比特

2. 基本数据类型

char、byte、int、short、long、boolean、float、double

3. 单精度和双精度

float 32 位浮点数 double 64 位浮点数

符号位 + 指数部分 + 尾数部分

单精度 1 符号位 + 8 指数位 + 23 尾数位

双精度 1 符号位 + 11 指数位 + 52 尾数位

4. int 和 char 类型互转

char 转 int 是自动类型转换

int 转 char 是强制类型转换

5. 包装器类型

Java 为基础数据类型提供了对应的包装器类型

而且由于自动装箱和自动拆箱机制的存在，不同数据类型之间的转换是自动的。

6. 引用数据类型

对象、接口、字符串

基本数据类型：

  1. 变量名指向的是基本数据
  2. 基本数据类型存在栈上

引用数据类型：

  1. 变量名指向的是对象的内存地址
  2. 对象存在堆上

7. 堆和栈

从速度上来说，栈要比堆快的多的多，但是同时，栈很小，他只能存基本数据类型，而引用数据类型，例如对象，它太大了，栈存不下。

所以我们在代码中 new 一个对象的时候，都是在堆里开辟一个空间进行保存，只会将一个指向堆的内存地址保存在栈中。

Java 数据类型转换

1. 自动类型转换

byte -> short -> int -> long -> float -> double

char -> int

以上是需要熟练记住的自动转换规则。

还有一种转换规则，编译器会自动帮我们转换在取值范围内的数据，比如说 byte b = 50; 在我们的理解下这是 50 是 int，理论上这样需要强制转换，但是因为 50 在 byte 类型的取值范围内，编译器隐性的帮我们转换了。

2. 强制类型转换

记住一点，强制类型转换要保证转换后的值的取值范围在新的类型的范围内，否则就会报错。

当前，强制类型转换肯定会导致精度的问题。

Java 基本数据类型缓存池

除了 Float 和 Double

其他的包装类都有一个常量缓存池 范围是 Byte 型的取值 -127-128

Boolean：true false

Character：\u0000 - \u007F

valueOf 的源码其实先判断是否在缓存池中，如果在直接从常量池中返回数值；如果不在就会 new 一个新的包装类对象。

Java 运算符

1. 算术运算符

加减乘除 + 取余 + 自增自减

2. 关系运算符

大于 and 小于 + 等于 and 不等于 + 大于等于 and 小于等于

3. 位运算符

其实位运算符挺好理解的，想一想二进制的排序，往左移 << 就是乘 2 的 几次方，往右移就是除以 2 的几次方

4. 逻辑运算符

&& 逻辑与

|| 逻辑或

! 逻辑非

逻辑与和逻辑或这两个运算符都有短路的效果，如果左侧条件不满足，则右侧条件不会执行。

5. 赋值运算符

6. 三元运算符

三元运算符主要是简单的替换 if-else 语句

int min=(a<b)?a:b;

如果 ? 前面的为 true 则执行 : 前面的 否则执行 : 后面的

Java 流程控制语句

  1. if-else
  2. switch case
  3. for
  4. while(do while)
  5. continue
  6. break

两数相加

这个题我的思路是看示例想的，因为题目中说了数字是逆序存在链表里的，其实就相当于省事了，正好是从个位开始计算。然后两个链表依次读取结点的值，然后加起来（结点 1 + 结点 2 + 进位）进位就是除以 10，最终结果就是模 10。

其实两个列表的长度不同好处理，满足两个有一个不为空整个加法继续，然后短的链表后边的空值统一设置为 0 就行，然后最后要考虑一下最高位的进位。

面渣逆袭

1. 什么是 Java

Java 是一个面向对象的编程语言，他继承了 C++的优点，同时又舍弃了 C++中容易出错的内容，比如指针和垃圾回收。

Java 的特点是一次编译多处运行，这主要是依靠于 JDK 实现的，因为 Java 在编译后会生成字节码，字节码会在 JVM 中运行，所以只要对应的平台安装了对应的 JDK，Java 代码就可以顺利执行。

2. Java 语言的特点

  1. Java 语言的三大特点，继承、封装、多态。
  2. 一次编译，多处运行。主要是和 Java 程序的运行原理有关，Java 程序在 JVM 中运行。
  3. 多线程，Java 内置多线程操作。
  4. 支持 JIT 即时编译，可以通过优化热点代码提升程序的运行速度。

3. JDK JRE JVM 有什么区别

JDK 是 Java 开发工具包，包含了 JRE（Java 运行环境）和 JVM（Java 虚拟机）

大多数情况下，如果只需要运行 Java 程序，那么就安装 JRE 就可以。

JVM 的任务是将 Java 字节码转换成对应平台的机器码。

4. 说说什么是跨平台 原理是什么

Java 程序的特点就是一次编译，多处部署，这使得它有良好的跨平台的特性，也就是说只需要写一份代码，就可以在多个平台上运行。

原理就是 Java 程序编译好的是 Java 字节码，然后 Java 程序运行的时候通过 JVM 转换成对应平台的机器码。

5. 什么是字节码 字节码的好处是什么

字节码就是 Java 程序编译后的 class 文件，好处就是通过 JVM 这个桥梁，一份字节码就能在不同的平台运行。

6. 为什么说 Java 是解释与编译并存的语言

编译型语言意味着一次性就将代码编译成某一个平台的机器码。

解释型语言则依靠着解释器逐行将代码翻译成对应平台的机器码。

之所以 Java 是解释与编译并存的语言，主要就是因为 Java 是将代码一次性编译成字节码，这一点很像编译型语言，但是 Java 又是通过 JVM 将代码逐行翻译成对应平台的机器码进行运行。
